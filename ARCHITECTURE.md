## Conventions

### Files and Naming

- **Files:** `kebab-case.js` (or `.jsx` for React components)
- **Directories:** `kebab-case/`
- **Components:** `PascalCase` function names, `kebab-case` file names
- **Functions:** `camelCase`
- **Constants:** `UPPER_SNAKE_CASE`

### Imports

- Use relative imports with `.js` extensions: `import { foo } from "./bar.js"`
- Import order: node built-ins, external packages, internal modules, relative imports

### Validation

- Use Zod for all external input validation
- Co-locate schemas with the code that uses them in `lib/contracts/`
- Schema naming: `FooSchema` for Zod objects, `Foo` for inferred types

### Error Handling

- Use result objects for expected failures: `{ ok: true, value } | { ok: false, error }`
- Throw only for unexpected/programmer errors
- Never swallow errors silently

### Testing

- Co-locate test files with source: `foo.test.js` next to `foo.js`
- Test behavior, not implementation
- Mock external dependencies, not internal modules
